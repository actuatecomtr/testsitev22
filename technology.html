The Platform

Actuate's Edge Platform is the operational layer between devices and downstream systems. It connects hardware from different manufacturers, processes data locally, and produces structured outputs — without requiring a single vendor ecosystem or continuous cloud connectivity.

The architecture is built around three functional layers.

Architecture

Three-layer design

Universal integration → on-edge processing → structured output and selective distribution.

Layer 1

Universal sensor and device integration

The first problem in any mixed-vendor environment is that devices do not share a common language. Industrial protocols, analog interfaces, mobility standards, and imaging systems all operate differently. The edge layer is designed to connect across all of them.

Supported interfaces

  • Analog: 4–20mA, 0–10V
  • Industrial protocols: Modbus RTU/TCP, CAN bus, OPC-UA, BACnet
  • Mobility: OBD-II, V2X
  • Imaging: RGB, thermal, multispectral (including NDVI)
  • Custom and legacy interfaces on a project-specific basis

A single deployment can ingest data from a PLC, a third-party field sensor, an energy analyser, and an imaging system simultaneously — within the same architecture, without replacing or reconfiguring existing hardware.

In the DHL Supply Chain Turkey deployment, this allowed existing warehouse infrastructure to be connected without hardware replacement — a hard requirement under DHL's security policy.

Layer 2

On-edge processing and AI

Data is processed locally — on embedded hardware at the deployment site, not on a remote server.

Local processing handles

  • Signal filtering and normalisation across mixed data types
  • Rule-based logic and threshold monitoring
  • Event and anomaly detection
  • Lightweight inference for constrained devices — including small-footprint models for low-power hardware that run classification and detection on-device without cloud dependency
  • Image-based event detection on Linux-class edge devices

The system runs across a wide compute spectrum — from bare-metal microcontrollers (ARM 32-bit, from 16KB, no OS) through Linux-based mini edge devices, MEC-class nodes, and midsize edge aggregation environments handling multi-source data for concurrent consumers.

Processing performance is determined by local compute capability, not network conditions. This matters in latency-sensitive environments and wherever data residency requirements prevent off-site processing.

Layer 3

Structured output and selective distribution

Raw data is rarely what downstream systems need. The output layer produces structured information rather than continuous streams:

  • Alerts and event logs
  • Aggregated metrics and KPIs
  • Flagged records or selected frames
  • Decision-ready outputs for dashboards, APIs, or institutional systems

Outputs can be routed to public cloud, private infrastructure, on-premise systems, or multiple endpoints simultaneously. In multi-stakeholder deployments, different consumers receive only the outputs relevant to their role, from the same physical source.

TRL positioning

Core integration and processing functions have been validated in operational environments across multiple sectors, corresponding to TRL 7–8. Sector-specific implementations — including the DHL warehouse deployment and the SafeRoute-6G pilots — have reached TRL 8–9.

Actuate also participates in projects at TRL 4–6, where the objective is extending the platform into new communication environments or application domains. The active Horizon Europe agricultural project operates in this range for certain components.
